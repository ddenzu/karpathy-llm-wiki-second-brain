# 📚 카파시 LLM Wiki (방법론 메모)

← 돌아가기: [[index]]

> 이 보관함을 **왜 이렇게 짓는지**의 출처. 위키가 자기 자신에 대한 지식도 갖게 됨.
> 운영 규칙은 루트 `CLAUDE.md` 참고.

## 한 줄 정의
**안드레 카파시**가 제안: LLM에게 코드만 짜게 하지 말고, **나만의 위키(지식베이스)를 짓고 유지보수하게 하라.**

## 핵심 원리 — RAG를 넘어서
- RAG: 질문마다 원본을 **매번 다시 검색** (지식 안 쌓임)
- LLM Wiki: LLM이 **미리 구조화된 위키를 지어둠** → **복리 효과** (원본 하나 넣으면 미래 모든 질문이 개선됨)
- 모든 주장은 원본으로 **역추적** 가능
- 요지: "읽기가 지루한 게 아니라 **정리가 지루하다**. 그 정리를 LLM이 한다."

## 표준 3계층
| 계층 | 역할 | 이 템플릿 대응 |
|---|---|---|
| `raw/` | 원본 (수정 금지·단일 진실 소스) | `raw/` 인박스 (+ 선택: 외부 심링크) |
| `wiki/` | LLM이 생성·유지하는 노트 | `wiki/` (LLM이 `/ingest`로 채움) |
| `CLAUDE.md` | 스키마 = AI를 규율 있는 관리자로 | 루트 `CLAUDE.md` |

## 운영 루프
- **Ingest**: 새 원본 → 요약 노트 → 관련 노트 연쇄 갱신 → 인덱스/로그 갱신
- **Query**: 인덱스→드릴다운으로 출처 단 답 (벡터 RAG 안 씀)
- **Lint**: 주기적으로 모순·고아 노트·빈 노드·낡은 주장 점검

## 함정 (꼭 피할 것)
- **스키마(CLAUDE.md) 없으면** 출력 제각각 → 위키 붕괴 ← 1순위 필수
- **범위 과다** → 좁혀라
- **raw 수정** → 추적 붕괴 (원본은 신성불가침)
- **위키만 덩그러니** → 방치. 반드시 **실제 워크플로우에 끼워넣기**

## 효과 조건
반복적으로 비슷한 주제를 다루는 사람(개발자·리서처)에게 강력.
